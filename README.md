### 简介
该软件用于将你的手机和电脑通过局域网连接 实现跨屏协作 并提供了实用的功能

### 特点
- 支持通过扫码连接和手动输入地址连接
- 绑定设备后 只需打开软件即可自动执行连接
- 所有传输均进行加密
- 控制Android端媒体播放
- 查看和管理Android端实时通知列表
- 远程下载手机上的文件 支持预览音视频和图片
- 通知转发记录可设置为需要验证才能查看 支持深层隐藏
- 软件窗口可设置不允许截录屏
### 下载
项目在Github Release发行 你可以在Release中(PC页面右侧)下载安装包

该软件涉及隐私数据操作 务必确保在官方渠道安装
### 使用方法
安装后直接启动即可 系统可能会弹出访问本地网络的提示框 务必选择允许(否则无法正常工作)

首次启动软件需要生成唯一的加密证书(该过程是全自动的) 耗时可能稍久

之后可通过Android端扫码二维码连接 如需手动连接 点击PC端下方的'轮盘'图标将显示IP 端口和本次的配对码 输入Android端对应位置即可

如Android设备没有安装客户端 可点击PC端连接页面下方的'下载'按钮并在Android设备上使用浏览器扫描新显示的二维码 可以在局域网内直接下载客户端安装包
### 核心功能
- 数据互传:在双端互相发送文件或文本消息
- 通知转发:将手机上的通知实时转发到计算机上并通过Windows的通知功能实时展示 同时保存进通知历史记录中
- 文件浏览:可以在PC端查看与下载手机上的文件 并支持对部分音视频和图片格式远程查看、播放
- 实时通知:实时查看Android设备上的通知列表 并支持远程清除通知
- 信任模式:影响连接后计算机可获取的数据类型 用于在部分情况下保护隐私 详见[Android端FAQS](https://github.com/NativeStar/SuishoConnector-Android/blob/master/FAQS.md)
- 音频转发:将Android设备上播放的音频传输到电脑上同步播放

### 杂项功能
- 媒体控制:获取通知权限后 可控制手机端当前的媒体播放状态 
- 通知锁定与隐藏:可设置锁定'通知转发'页面 避免他人直接查看到转发的通知记录 同时支持深度隐藏功能 详见'设置'章节
- 媒体预览:支持远程预览手机上的图片和音视频文件 其中音频文件支持显示内嵌歌词
- 保护方式设置:支持通过WindowsHello(生物识别)或密码保护通知转发页面 未来可能支持更多页面保护
- 阻止截屏录屏:开启后软件窗口将无法被截屏和录屏捕获到(变为全透明或全黑 取决于Windows版本)
- 满电提醒:当手机充满电时在PC端发出通知提醒
### 设置
#### 通知展示方式
- 全部展示:默认模式 会展示通知的标题、内容、发送的应用名
- 仅应用名:只展示通知发出的应用名 内容变为'收到来自'***'的通知'
- 全部隐藏:只弹出通知 但不包含任何实际信息 内容变为'连接的手机收到一条通知'
- 不推送:不再实时推送被转发的通知 但仍会记入历史
##### 主页设置
- 绑定/解绑设备:对当前连接的设备进行绑定(如当前连接和绑定设备一致则解绑 否则换绑) 绑定成功后 在连接页面时绑定的设备间将可以自动进行连接
- 掉线轮询间隔:设置速度越快则当连接异常中断时弹出掉线提醒的反应越快 但可能提升手机耗电量
- 启用通知转发:关闭后将不再处理任何通知转发请求 包括记录通知历史
- 使用通知历史记录:将经过转发的通知全部进行记录 用于在'通知转发'页面中查询
- 默认通知展示方式:当通知发起的应用无独立配置且通知内容未被黑白名单拦截时遵循的展示方式 具体参考'通知展示方式'章节
- 验证方式:用于保护部分隐私页面(暂时仅用于'通知转发'页) 支持WindowsHello(生物识别等) 或软件内输入密码
- 通知转发记录保护:开启后 查看'通知转发'记录和打开设置面板前需要验证
- 截录屏保护:开启后软件窗口将无法被截屏和录屏捕获到(变为全透明或全黑 取决于Windows版本) 适用于直播、需要共享屏幕的视频会议等(注意右下角弹出的通知无法受到保护!)
#### 通知转发过滤配置
'通知转发'点击上方漏斗图标可打开此窗口
- 内容过滤:分为黑白名单模式 黑名单模式将阻止通知中含有指定内容的通知被推送和记录 白名单反之(只允许通知中含有指定内容的通知被推送和记录)
##### 应用配置
下面所有设置项均为针对单个应用的配置
- 启动通知转发:是否接收来自该应用的通知 关闭后将彻底拒绝该通知(也不会被计入历史)
- 启用单独配置:启用后对该应用通知的处理不再遵循默认和全局配置 如'默认通知展示方式'
- 进行内容过滤:关闭后来自该应用的通知将不再通过黑白名单处理(特赦)
- 深层隐藏:开启后该应用的通知将不直接在历史通知列表中显示 需要在'通知转发'页使用鼠标右键点击解锁图标并通过验证后才会显示(需要配合'通知转发'页保护使用)
- 不保存至推送记录:无论全局设置是否保存记录 开启该选项后来自该应用的通知必定不会被保存
- 通知展示模式:和全局设置类似 但只针对一个软件 具体参考'通知展示方式'章节
### 那些可能被忽略的功能 
- 当处于'数据互传'或'通知转发'页面时 再次点击选项卡可回到列表底部
- 你可以将文件拖动到'数据互传'页面的列表上或应用程序的桌面图标上进行快捷上传
- 在'数据互传'或'通知转发'页面内可以通过Ctrl+F快捷键触发搜索(虽然也有对应的按钮触发搜索)
- 在'数据互传'页面中右键文本消息可进行全部复制 如果消息为纯URL可右键快捷打开 如果为包含URL的其他数据可选择URL后右键执行'打开选中链接' 右键文件消息可在资源管理器中查看
- 在'通知转发'中右键通知项可快捷复制通知内容或打开对应程序的转发设置
- '文件浏览'页面中右键文件夹可将其收藏 右键收藏的文件夹可取消收藏
### 开发
项目基于Electron开发 使用Electron-Forge封包 通过npm管理依赖

主进程代码位于'renderer'目录下 通过pnpm管理依赖 使用React+Vite实现

本地部署方法(在项目根目录打开PowerShell)
```
# 安装主进程依赖
npm run i
# 切换至渲染进程
cd ./renderer
# 安装渲染进程依赖
npm i
# 拉起渲染进程调试
npm run dev
# 返回主进程
cd ../
# 拉起主进程调试
npm run dev
```
构建方法(部署完成后在项目根目录打开PowerShell)
```
# 根据需求选一个执行即可

# 以portable模式生成 可直接运行
npm run package
# 创建exe安装包
npm run make
```
项目基于MIT协议开源
